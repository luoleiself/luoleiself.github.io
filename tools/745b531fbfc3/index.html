发表于 2021-04-17 更新于 2021-04-20 分类于 tools

配置篇

Git 仓库用户凭据

场景: 使用 https 方式与远程仓库同步时, 不想每次都在提示框中输入用户名和密码确认, 此方式在用户家目录下创建(修改) .git-credentials 文件, 存储用户名和密码

Git 环境定制

  • local 读取仓库配置文件 .git/config

  • global 读取用户配置文件 .gitconfig

unset 重置
add 添加配置
get 获取配置
alias 命令缩写
alias.log 美化

  • %H commit hash

  • %h commit short hash

  • %T tree hash

  • %t tree short hash

  • %P parent hash

  • %p parent short hash

  • %a[n|N] 作者名字

  • %a[e|E] 作者邮箱

  • %a[d|D|r|t|i] 日期格式

  • %c[n|N] 提交者名字

  • %c[e|E] 提交者邮箱

  • %c[d|D|r|t|i] 提交的日期格式

  • %d ref 名称

  • %e encoding

  • %s commit 信息标题

  • %f 过滤 commit 信息的标题使之可以作为文件名

  • %b commit 信息内容

  • %N commit notes

  • %g[D|d] reflog selector

  • %gs reflog subject

  • %Cred 切换至红色

  • %Cgreen 切换至绿色

  • %Cblue 切换至蓝色

  • %Creset 重设颜色

  • %C(color) 指定颜色

  • %n 换行

  • %m left right or boundary mark

  • %%a raw %

  • %x00 print a byte from a hex code

  • %w([[,[,]]]) switch line wrapping, like the -w option of git-shortlog(1).

remote 设置
add 添加
remove 删除
rename 修改关联仓库名称
set-url 设置关联仓库地址

  • push 设置推送 url

  • add 保持当前的 url,并添加一个新的 url

  • delete 将删除匹配到的 url, 并添加一个新的 url

get-url 获取关联仓库地址

  • push 获取推送 url

  • all 获取所有 url
