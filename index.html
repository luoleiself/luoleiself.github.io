操作篇

初始化仓库

克隆仓库

分支

查看

  • a|all 列出所有本地和关联远程分支

  • r 列出关联远程分支

  • v|verbose 列出分支并显示当前提交信息摘要

新建
移动|修改

  • m 移动|修改分支, old-branch 无则为移动操作

  • M 强制移动|修改分支即使新分支存在, old-branch 无则为移动操作

复制

  • c 复制分支和分支提交历史

  • C 强制复制分支和分支提交历史

删除

  • d 删除本地分支,一般和 r 配合删除关联远程分支

  • D 强制删除本地分支, 即使分支未被合并

切换
切换并创建新分支

基于远程分支创建新分支,自动建立追踪关系

分支追踪关系
  • t|no-track 建立|取消分支追踪关系
  • u|unset-upstream 建立|取消分支追踪关系
当前分支和远程分支建立追踪关系
指定分支和远程分支建立追踪关系
分支追踪关系,提交摘要

分支合并
merge
  • fast-forward(ff): 快速合并, 不创建新的 commit, 原分支删除后提交记录消失, 默认方式
  • no-ff: 不快速合并, 保留原有分支记录, 创建新的 commit
  • squash: 合并一些不必要的 commit, 创建新的 commit
  • stat: 合并结束后统计显示区别
  • continue: 解决冲突后结束合并
  • abort: 中断解决冲突结束合并
  • quit: 放弃合并

rebase
  • i|interactive: 交互式操作
  • continue: 解决冲突后结束合并
  • abort: 中断解决冲突结束合并
  • quit: 放弃合并
  • skip: 重启合并跳过当前的修改

选择合并

  • 选择一个或者多个 commit, 合并进当前分支, 手动 commit

  • 选择 commit 区间合并, 含尾不含头

  • 选择 commit 区间合并, 包含头和尾

文件操作

添加暂存区
  • all|A|. 提交所有文件
  • i|interactive 交互式操作
  • n|dry-run 不执行任何操作, 只显示做什么
撤销文件

  • 撤销工作区文件的变更

  • 恢复上一个 commit 的所有文件到工作区

  • 恢复指定 commit 的指定文件到工作区

提交
  • a|all git add 的缩写
  • m|message commit 的注释
  • amend 改写上一次 commit 的注释
版本变更
reset

  • mixed: 还原版本库和暂存区, 工作区保持不变, 默认方式

  • soft: 还原版本库, 暂存区和工作保持不变

  • hard: 还原版本库和暂存区和工作区

  • keep: 还原版本库和暂存区, 并更新工作区中的 commit 和 HEAD 之间不同的文件, 如果不同的文件本地有更改则中止

  • merge: 还原版本库和暂存区, 并更新工作区中的 commit 和 HEAD 之间不同的文件, 但保留暂存区和工作区中不同的文件(既有未添加的更改), 如果不同的文件有未暂存的更改则中止

revert

撤销一个或多个 commit, 并手动提交 commit

配置篇

概念

Remote

远程仓库, 托管代码的服务器, 团队协作开发时都与此进行同步

Repository

本机仓库(或版本库), 管理所有被提交的代码版本, 其中 HEAD 指向最新放入仓库的版本

Index / Stage

暂存区, 临时存放改动的需要被提交的文件列表信息

Workspace

工作区, 当前可见的随时可以进行操作的区域

Git 仓库用户凭据

场景: 使用 https 方式与远程仓库同步时, 不想每次都在提示框中输入用户名和密码确认, 此方式在用户家目录下创建(修改) .git-credentials 文件, 存储用户名和密码

  • 清除本地存储的用户名和密码凭据

  • 存储凭据, 在第一次 push 时提示输入

Git 环境定制

  • local 读取仓库配置文件 .git/config
  • global 读取用户配置文件 .gitconfig, 不加默认为仓库配置
